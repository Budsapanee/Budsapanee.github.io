The Problem

This project was created to improve the user experience of batch payments.

Operating expenses are common while running a business, and some of those expenses must be paid on a regular basis. For example, workplace phone bills. There may be many mobile phones, but they all use the same service provider, which should be listed on the same expense document. However, the system no longer supports this function. As a result, it does not perform as well as it should.

My Role

I took on a UX designer role in this project. My job is to understand and observe our users, which includes business owners and accountants.

Then, make a prototype to test with users and finish the design to get ready for implementation.

Challenge

There are lots of options for creating batch payment documents. The challenge is to avoid breaking any system conventions.
